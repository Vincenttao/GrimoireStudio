## 1. 目标用户画像

### 1.1 主要用户群体：网文作者

| 维度 | 描述 |
|------|------|
| **职业身份** | 全职/兼职网文作者，连载平台签约作家 |
| **年龄范围** | 20-40 岁 |
| **技术素养** | 中等 — 会用 Word/Git，但非程序员 |
| **收入水平** | 中等 — 对工具成本敏感，追求性价比 |
| **写作经验** | 1-10 年，有完整作品经验 |

### 1.2 技术素养细分

| 层级 | 比例 | 特征 |
|------|------|------|
| **高阶用户** | 20% | 懂 Git、会用 API、可本地部署 |
| **中阶用户** | 50% | 会用图形界面、理解 LLM 概念、需要引导 |
| **入门用户** | 30% | 仅会用现成工具、需要详细教程 |

## 2. 用户需求与痛点

### 2.1 核心痛点排序

| 痛点 | 严重程度 | 频率 | 解决方案 |
|------|----------|------|----------|
| 角色崩坏 (OOC) | 🔴 高 | 高 | Grimoire + 快照链 |
| 节奏拖沓 | 🔴 高 | 中 | Maestro 张力评分 |
| 文风平庸 | 🟡 中 | 高 | Camera Agent |
| 大纲维护 | 🟡 中 | 中 | 自动生成 |
| 记忆管理 | 🟢 低 | 低 | 滑动窗口 |

### 2.2 期望工作流

1. **快速启动** — 无需写大纲即可开始
2. **实时反馈** — 看到推演过程，可控可调
3. **低成本** — Token 消耗透明可预测
4. **数据安全** — 作品数据在本地，不泄露

## 3. 会话模式分析

### 3.1 典型使用场景

| 场景 | 时长 | 频率 | 期望产出 |
|------|------|------|----------|
| 冷启动创作 | 30-60 分钟 | 首次 | 完成第一章设定 |
| 日常连载 | 2-4 小时 | 每天 | 2000-5000 字正文 |
| 大纲调整 | 30 分钟 | 每周 | 章节结构修改 |
| 全书渲染 | 1-2 小时 | 完结时 | 全书风格统一 |

### 3.2 Token 预算预期

| 活动类型 | 预估 Token | 成本估算 (GPT-4) |
|----------|-----------|------------------|
| 单章推演 | 50k-100k | $1-3 |
| 一本短篇 (5章) | 300k-500k | $6-15 |
| 长篇连载 (100章) | 5M-10M | $100-300 |
