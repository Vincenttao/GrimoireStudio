# 👤 Genesis Engine: 用户画像文档

**版本:** v2.0 (Web Novel Author Edition)
**更新日期:** 2026-04-24

---

## 0. 北极星用户（North Star）

**中文网文连载作者。** 没有次要人群。所有 UI 默认值、Muse 语汇、预设文风、渲染参数、评估指标，全部以"能帮一个网文作者每天稳定产出 3000+ 字"为第一优先。

> 其他潜在用户（剧本杀写手、TRPG KP、同人作者、轻小说作者）在架构上不被阻挡，但**不作为任何功能的设计输入**。扩展他们的需求留给社区或 V3.0 后。

## 1. 用户画像

### 1.1 核心人群特征

| 维度 | 描述 |
|------|------|
| **职业身份** | 全职或半职连载作者；主要平台：起点/番茄/晋江/纵横/七猫 |
| **年龄范围** | 22-40 岁 |
| **技术素养** | 中低 — 会用 Word/WPS，会调接口的是少数 |
| **收入水平** | 月入 2k-5w 不等，对工具成本极度敏感 |
| **写作经验** | 1-10 年，至少有一本已完结或连载中的作品 |
| **硬件环境** | Windows / macOS 个人电脑；少数用平板写作 |

### 1.2 技术素养细分

| 层级 | 比例 | 特征 |
|------|------|------|
| **高阶作者** | 15% | 会调 OpenAI API、能本地部署、读得懂 JSON |
| **中阶作者** | 55% | 会用图形界面、理解 LLM 概念、需要上手引导 |
| **入门作者** | 30% | 仅会用现成 SaaS 工具、需要一键安装包 |

### 1.3 主要平台分层（V1.0 必须支持平台预设）

| 平台 | 主流读者偏好 | Render Mixer 默认值 |
|------|------|------|
| **起点中文网** | 爽文 / 打脸 / 热血 | Subtext=0.2, Style="热血爽文", max_sent_len=30 |
| **番茄小说** | 短句 / 节奏快 / 爽点密集 | Subtext=0.15, Style="快节奏爽文", max_sent_len=20 |
| **晋江文学城** | 情感 / 细腻 / 心理描写 | Subtext=0.6, Style="江南烟雨", max_sent_len=40 |
| **纵横中文网** | 权谋 / 军事 / 历史 | Subtext=0.3, Style="肃杀权谋", max_sent_len=35 |
| **七猫 / 飞卢** | 短平快 / 流水账爽文 | Subtext=0.1, Style="白开水爽文", max_sent_len=18 |

## 2. 用户需求与痛点

### 2.1 核心痛点排序（网文特化）

| 痛点 | 严重度 | 频率 | 解决方案 |
|------|--------|------|----------|
| **卡文 / 今天不知道写啥** | 🔴 极高 | 每周 2-3 次 | `[卡文救急]` 按钮 + Spark 灵感生成 |
| **角色 OOC（说话不像自己）** | 🔴 极高 | 每章 | Grimoire + VoiceSignature（口头禅/称谓） |
| **章节字数卡不准（水字数/凑字数）** | 🔴 极高 | 每章 | Camera `target_char_count` 约束 + 水字数检测 |
| **章节末尾钩子太平** | 🔴 高 | 每章 | Ending Hook Guard（结尾重渲染） |
| **爽点节奏飘（连续铺垫读者跑了）** | 🟡 高 | 每周 | 爽点节奏表 + Muse 主动提醒 |
| **节奏拖沓 / 流水账** | 🟡 高 | 每章 | Maestro `beat_type` 判据 |
| **文风平庸 / AI 味重** | 🟡 中 | 每章 | Camera + 平台预设 + 参考文段锚点 |
| **找不到之前写过的某个场景** | 🟡 中 | 每周 | Archive 全文检索（FTS5） |
| **大纲维护成本** | 🟢 中 | 每周 | Muse 自动生长 Grimoire |
| **断更焦虑（心理压力）** | 🟢 中 | 日常 | 日更连胜 UI + 字数进度条 |

### 2.2 用户不会明说、但必然违背的期望（Hidden Contracts）

这些是**作者默认产品应该做到、但需求文档里没人会写**的东西：

1. **改 HTML 不掉记忆** — 作者随手改错别字，不应该触发 Scribe 重新跑。
2. **关闭电脑重开接得上** — Scratchpad 必须落盘；推演 20 turn 崩了不能全丢。
3. **Token 成本可预测** — 开章前知道 "这章大概 $0.3"，不是用完才知道。
4. **不需要读文档就会用** — Muse 是唯一学习曲线，产品其他部分零门槛。
5. **和已有作品兼容** — 从 Word/txt 导入前 30 章也能接着写，不强迫从头构建 Grimoire。

### 2.3 网文日更生态的关键词汇（产品语汇必须吻合）

Muse、Settings、Monitor 里出现的措辞都要用网文作者的语言，不要用文学/软件工程术语：

| 网文术语 | 含义 | 产品里的对应 |
|------|------|------|
| **金手指** | 主角独有优势/系统/外挂 | 角色的隐藏动机/能力字段 |
| **爽点** | 读者感到爽的瞬间 | BeatType = `PAYOFF` / `SHOW_OFF_FACE_SLAP` |
| **打脸** | 反派被主角挫败的瞬间 | BeatType = `SHOW_OFF_FACE_SLAP` |
| **钩子** | 章末悬念 | Ending Hook Guard 检测对象 |
| **断更** | 错过日更 | 连胜计数器重置 |
| **水字数** | 无意义填充字数 | Padding Detector 警告 |
| **本章说** | 读者段落评论 | 模拟读者反馈（V2） |
| **卡文** | 写不出 / 不知道写啥 | `[卡文救急]` 按钮 |
| **存稿** | 提前写好的章节 | Chapter 的未发布状态 |
| **均订** | 平均订阅数 | (产品不涉及，仅术语兼容) |
| **高能章** | 爽点密集的关键章节 | 爽点节奏表的热度标记 |
| **太监** | 半途弃坑 | (避讳用语，产品不出现) |

## 3. 会话模式分析

### 3.1 日更作者一天的典型使用流程

```
09:00  打开项目 → Muse 问候："昨天写到第 47 章，宁毅刚离开道观。今天想怎么开场？"
09:05  作者回复或点 [接着写] → Muse 生成 3 个 Spark 候选
09:10  作者选一个 → 推演启动 → Monitor 实时看角色博弈（15-30 分钟）
09:40  [Commit Spark 1] → 完成本章 Scene 1
09:45  追加 Scene 2（高潮/钩子）→ 再推演一轮
10:15  Render All → Camera 按平台预设渲染 → 字数 2950/3000 ✅
10:20  作者手动润色 3-5 处 → 钩子守卫通过 → [Commit 第 47 章]
10:25  模拟读者反馈："老白 / 爽文党 / 剧情党" 3 条评论
10:30  连胜 +1 🔥 → 关闭项目
```

**关键指标**：从打开到 Commit，**60-90 分钟**内应能稳定完成一章 3000 字。超过 2 小时说明产品有摩擦。

### 3.2 典型场景分布

| 场景 | 时长 | 频率 | 期望产出 |
|------|------|------|----------|
| **日更一章** | 60-90 分钟 | 每天 | 3000-5000 字 |
| **卡文救急** | 5-10 分钟 | 每周 2-3 次 | 3 个 Spark 方向 |
| **冷启动新书** | 1-2 小时 | 开新书时 | Grimoire 骨架 + 第 1 章 |
| **大纲调整** | 30 分钟 | 每周 | 后续 3-5 章走向 |
| **全本渲染** | 2-4 小时 | 完本时 | 统一文风的全本 epub/txt |

### 3.3 Token 预算（基于真实网文作者成本承受力）

**默认模型组合（V1.0 推荐）**：
- Muse / Maestro / Scribe / 规则判定：Qwen-turbo 或 DeepSeek-chat（¥1-2 / 百万 token）
- Character / Camera：Qwen-plus / DeepSeek-reasoner（¥5-20 / 百万 token）

| 活动 | 预估 Token | 成本（Qwen+DS 组合） | 成本（GPT-4.6） |
|------|-----------|------|------|
| 单章推演 + 渲染 | 30k-60k | **¥0.3-0.8** | $1-3 |
| 一本短篇 (5 章) | 200k-400k | **¥2-5** | $6-15 |
| 长篇连载（100 章） | 3M-6M | **¥30-80** | $100-300 |

> **硬指标**：单章 Token 成本必须控制在 **¥1 以内**，否则作者算完账就会流失。Settings 里必须有前瞻式成本预测。

## 5. 反 Persona（明确不服务的人群）

这些人群**不是**目标用户，产品不为他们让步：

| 不服务人群 | 原因 |
|-----------|------|
| **英文小说作者** | 所有 Prompt / Style / 平台预设均中文特化 |
| **剧本创作者** | 格式诉求根本不同（对白/指示/动作分离） |
| **学术/非虚构** | 不需要 Character Agent 博弈 |
| **纯文学严肃小说** | 不需要爽点节奏、钩子守卫、平台预设 |
| **多人协作团队** | 架构固化为单用户单体，分享靠 sqlite 文件 |
| **企业 / 内容工作室** | 无账号体系、无多租户 |

## 6. V1.0 → V1.1 → V2.0 的用户画像演进

| 维度 | V1.0（核心引擎） | V1.1（网文作坊） | V2.0（造梦控制台） |
|------|-----------|-----------|-----------|
| 目标作者 | 能本地跑 Python 的高阶作者 | **中阶作者（主力人群）** | 入门作者（零门槛） |
| 必会技能 | 改 .env / 读日志 | 会和 Muse 对话 | 完全无 |
| 学习曲线 | 陡 | 平缓 | 零 |
| 核心验收 | 推演能跑通 | **连续日更 7 天不断更** | 一对话产一章 |
| 默认平台预设 | 无 | 起点 | 起点 + 番茄 + 晋江 |

## 7. 用户不会容忍的三件事（产品死线）

1. **崩了丢稿**：推演到一半崩了，之前的内容全没了。直接卸载。
2. **角色 OOC**：宁毅说出"尊敬的用户您好"。公开吐槽/发微博。
3. **字数算不准**：要 3000 字给 2200 字，或者水到 4500 字。流失。

任何版本发布前必须保证这三条都不出现。
