**版本:** v1.0

## 1. 流程概述

冷启动流程是指用户首次使用 Genesis Engine 时，从"空白项目"到"产出第一章正文"的完整引导过程。The Muse Agent 作为全流程的引导者。

## 3. 详细阶段规范

### 阶段 1: 灵感碰撞 (Brainstorm)

**触发:** 用户首次进入系统或点击"新项目"

**Muse 行为:**
```
System Prompt (简化版):
你是 The Muse，创世引擎的责编。用户是新作者，请引导他：
1. 询问他想写什么类型的故事
2. 帮他确认主角和核心冲突
3. 提出建议但尊重他的选择
```

**对话示例:**
```
用户: 我想写个赛博朋克修仙

Muse: 🎭 很有趣的想法！赛博修仙结合了科技与灵性。让我确认几个关键点：

1. **主角是谁？** 一个废柴？天才？还是复仇者？
2. **核心冲突是什么？** 门派争斗？资源争夺？还是身份危机？
3. **你希望读者感受到什么？** 热血？悬疑？还是讽刺？

用户: 主角是个搞笑道士，用赛博义体修仙

Muse: ✨ 太棒了！让我为你创建这个角色...

[tool_call 卡片出现]
{
  "action": "create_entity",
  "payload": {
    "type": "CHARACTER",
    "name": "李道长",
    "base_attributes": {
      "personality": "幽默散漫、实则深藏不露",
      "core_motive": "用科技证明传统修仙已经过时",
      "background": "前修仙宗门弟子，因理念不合出走..."
    }
  }
}

[确认创建] [修改设定]
```

---

### 阶段 2: 世界观固化 (Grimoire Injection)

**触发:** 用户点击"确认创建"

**后端行为:**
1. 调用 `POST /api/v1/grimoire/entities` 创建实体
2. 创建初始 GrimoireSnapshot
3. 更新左侧设定树

**Muse 后续引导:**
```
Muse: ✅ 已创建角色「李道长」

现在我们需要几个关键要素才能开始推演：
- 📍 一个场景（故事发生在哪？）
- ⚔️ 一个对手或障碍（谁/什么阻挡他？）

你想先设定哪个？
```

**最小可推演条件:**
- 至少 1 个 CHARACTER 类型实体
- 至少 1 个 LOCATION 类型实体 (或系统自动创建默认场景)

---

### 阶段 3: 大纲生成 (Storyboard Generation)

**触发:** 用户确认至少 1 个角色和 1 个场景

**Muse 行为:**
```
Muse: 🎬 万事俱备！让我为你生成第一章的冲突点：

**第一章：义体道观的秘密**

李道长收到一封神秘信件，要求他在三天内交出"赛博金丹"。但这件宝物根本不存在——他只是个骗子...

[生成 Spark]

{
  "action": "start_spark",
  "payload": {
    "user_prompt": "李道长收到神秘威胁信，要求交出不存在的赛博金丹。他决定将计就计，假装有这件宝物，实则在暗中调查幕后黑手。"
  }
}

[开始推演 ▶] [修改剧情]
```

## 4. 前端交互规范

### 4.1 引导 UI 元素

| 元素 | 位置 | 行为 |
|------|------|------|
| 欢迎卡片 | Muse Panel 顶部 | 首次进入显示，可关闭 |
| 进度指示器 | 左栏顶部 | 显示冷启动进度 (1/3, 2/3, 3/3) |
| 空状态提示 | 中栏 | 引导用户通过 Muse 开始 |

### 4.2 冷启动状态追踪

```typescript
interface ColdStartState {
  phase: 'brainstorm' | 'worldbuild' | 'outline' | 'complete';
  has_character: boolean;
  has_location: boolean;
  has_spark: boolean;
  completed_at?: Date;
}
```

---

## 5. 异常处理

| 异常 | 恢复方式 |
|------|----------|
| 用户中途退出 | 下次进入从上次断点继续 |
| 实体创建失败 | Muse 提示重试，保留对话上下文 |
| 用户不知道写什么 | Muse 提供 3 个随机创意种子 |

---

## 6. 完成条件

冷启动流程完成的标志：
- ✅ 至少 1 个 Character 实体
- ✅ 至少 1 个 Location 实体
- ✅ 至少 1 个 StoryNode (章节)
- ✅ 至少 1 次推演启动

完成后，隐藏引导 UI，进入正常工作流。
